**A. Add @app/* path alias to tsconfig.json**

In `tsconfig.json` under `compilerOptions`, add or merge:
```json
"baseUrl": "./",
"paths": {
  "@app/*": ["src/app/*"]
}
```
Keep all existing `compilerOptions` (strict, noImplicitOverride, etc.) — only add these two keys.

Verify `tsconfig.spec.json` extends `tsconfig.json` (it should by default from `ng new`). Open `tsconfig.spec.json` and confirm `"extends": "./tsconfig.json"` is present. If `paths` is NOT inherited, add explicitly to `tsconfig.spec.json` compilerOptions as well:
```json
"paths": {
  "@app/*": ["src/app/*"]
}
```
Run `npx tsc --noEmit` to confirm the alias does not break compilation.

**B. Configure commitlint**

Create `commitlint.config.js` at the repo root. Use ES module format ONLY if `package.json` has `"type": "module"`. If Angular's `ng new` does NOT set `"type": "module"` (it typically does not), use CommonJS:
```js
// commitlint.config.js (CommonJS — default for Angular projects)
module.exports = {
  extends: ['@commitlint/config-conventional'],
};
```
Check `package.json` for `"type": "module"` before deciding. If `"type": "module"` is present, use `export default { extends: ['@commitlint/config-conventional'] }` syntax instead.

**C. Wire the commit-msg husky hook**

Husky init (Task 1) creates `.husky/pre-commit`. We need `.husky/commit-msg` for commitlint (pre-commit runs before the message is entered — $1 is not available there):
```bash
echo 'npx --no -- commitlint --edit "$1"' > .husky/commit-msg
chmod +x .husky/commit-msg
```
Do NOT delete `.husky/pre-commit` (leave it, even if empty — it does no harm).

**D. Run OXfmt idempotency check**

```bash
# Check OXfmt config — run help to see supported config file names
npx oxfmt --help 2>&1 | grep -i config || true

# Format once
npx oxfmt src/**/*.ts

# Format again and diff — output must be identical (idempotency)
npx oxfmt src/**/*.ts
```
If re-running produces changes, restrict the glob to exclude problematic files and document the restriction. For Phase 1 scope, `src/**/*.ts` is sufficient.
